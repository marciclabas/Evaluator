Aquest programa emula una plataforma de laboratoris de programació. Els usuaris registrats a la plataforma es poden insciure en diferents cursos i enviar-\/hi problemes. Cada curs conté un conjunt de sessions, les quals contenen un conjunt de problemes.\hypertarget{index_autotoc_md3}{}\doxysubsubsection{Arquitectura de dades}\label{index_autotoc_md3}
El programa està organitzat en 4 classes úniques principals. Una col.\+lecció de problemes ({\ttfamily \mbox{\hyperlink{class_problem_collection}{Problem\+Collection}}}), un repositori de sessions (\mbox{\hyperlink{class_session_repository}{Session\+Repository}}), un conjunt de cursos ({\ttfamily \mbox{\hyperlink{class_course_set}{Course\+Set}}}) i un conjunt d\textquotesingle{}usuaris ({\ttfamily \mbox{\hyperlink{class_user_set}{User\+Set}}}), cadascun d\textquotesingle{}ells contenint tots els elements del tipus corresponent disponibles dins la plataforma, i.\+e., problemes ({\ttfamily \mbox{\hyperlink{class_problem}{Problem}}}s), sessions ({\ttfamily \mbox{\hyperlink{class_session}{Session}}}s), cursos ({\ttfamily \mbox{\hyperlink{class_course}{Course}}}s) i usuaris ({\ttfamily \mbox{\hyperlink{class_user}{User}}}s), respectivament.\hypertarget{index_autotoc_md4}{}\doxysubsubsection{Arquitectura de funcionalitats}\label{index_autotoc_md4}

Project within the PRO2 subject of the Informatics Engineering Degree at the Faculty of Informatics in Barcelona (UPC)\hypertarget{index_autotoc_md6}{}\doxysubsubsection{Abstract}\label{index_autotoc_md6}
This program mimics a programming lab like platform. Users signed in the platform can enroll in courses and submit problems to the platform. Each course contains several sessions which, in turn, contain several problems.\hypertarget{index_autotoc_md7}{}\doxysubsubsection{Data architecture}\label{index_autotoc_md7}
The program data is structured in 4 main singleton classes. A {\ttfamily \mbox{\hyperlink{class_problem_collection}{Problem\+Collection}}}, a {\ttfamily \mbox{\hyperlink{class_session_repository}{Session\+Repository}}}, a {\ttfamily \mbox{\hyperlink{class_course_set}{Course\+Set}}} and a {\ttfamily \mbox{\hyperlink{class_user_set}{User\+Set}}}, each of them containing all elements of their kind available in the platform, i.\+e., {\ttfamily \mbox{\hyperlink{class_problem}{Problem}}}s, {\ttfamily \mbox{\hyperlink{class_session}{Session}}}s, {\ttfamily \mbox{\hyperlink{class_course}{Course}}}s and {\ttfamily \mbox{\hyperlink{class_user}{User}}}s, respectively.\hypertarget{index_autotoc_md8}{}\doxysubsubsection{Functionality architecture}\label{index_autotoc_md8}
The program functionality is controlled through the standard input/output. All available commands ({\ttfamily System\+Command}s) are defined within the {\ttfamily io {\itshape namespace}, and all the handling logic of such commands is implemented in the {\itshape main} function. }
